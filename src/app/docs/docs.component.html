Overview
OAuth2 / OIDC browser flow For SPAs and browser applications

Use the standard authorization code flow with PKCE and OpenID Connect identity tokens.

  • Unauthenticated authorization requests redirect to /login.
  • Login establishes a Spring Security web session.
  • The saved authorization request resumes after successful login.
JSON JWT API For server-owned administration endpoints

Use the server's own REST authentication endpoints under /auth. This is separate from the OIDC authorization-code flow.

  • POST /auth/login returns an API access token.
  • The refresh token is an HTTP-only cookie scoped to /auth.
  • Protected requests use Authorization: Bearer ....

Quickstart
  1. 1

    Create a verifier and challenge

    Generate a high-entropy verifier and derive its SHA-256 code challenge.

const crypto = window.crypto;

function base64UrlEncode(buffer) {
  return btoa(String.fromCharCode(...new Uint8Array(buffer)))
    .replace(/\+/g, "-")
    .replace(/\//g, "_")
    .replace(/=+$/, "");
}

const verifierBytes = new Uint8Array(32);
crypto.getRandomValues(verifierBytes);

const codeVerifier = base64UrlEncode(verifierBytes);
const challengeBytes = await crypto.subtle.digest(
  "SHA-256",
  new TextEncoder().encode(codeVerifier)
);
const codeChallenge = base64UrlEncode(challengeBytes);
  1. 2

    Redirect to /oauth2/authorize

    Persist state, nonce, and the verifier before navigating away.

GET http://localhost:8080/oauth2/authorize
  ?response_type=code
  &client_id=admin-ui
  &redirect_uri=http%3A%2F%2Flocalhost%3A3000%2Foauth%2Fcallback
  &scope=openid%20profile%20email
  &state={random-csrf-state}
  &nonce={random-oidc-nonce}
  &code_challenge={codeChallenge}
  &code_challenge_method=S256
Without a Spring Security session, IAM Server redirects to /login. After login, the saved authorization request resumes automatically.
  1. 3

    Handle the callback

    Reject the response unless returned state matches the stored state exactly.

http://localhost:3000/oauth/callback
  ?code={authorization-code}
  &state={random-csrf-state}
  1. 4

    Exchange the code for tokens

    Send the original verifier. Public clients do not send a client secret.

curl -X POST http://localhost:8080/oauth2/token \
  -H "Content-Type: application/x-www-form-urlencoded" \
  -d "grant_type=authorization_code" \
  -d "client_id=admin-ui" \
  -d "redirect_uri=http://localhost:3000/oauth/callback" \
  -d "code={authorization-code}" \
  -d "code_verifier={original-code-verifier}"
  1. 5

    Receive the token response

    Keep access tokens in memory and validate the ID token before trusting claims. 

{
  "access_token": "eyJ...",
  "scope": "openid profile email",
  "id_token": "eyJ...",
  "token_type": "Bearer",
  "expires_in": 300
}

ID token checklist
  • Validate the RSA signature with /oauth2/jwks.
  • Validate iss against the configured issuer.
  • Validate expiration and issued-at times.
  • Validate nonce when one was sent.
Access token checklist
  • Send the token using the Bearer authorization scheme.
  • Use OIDC tokens only with their intended resource endpoints.
  • Use JSON API tokens only with this server's protected REST endpoints.
Server-owned JWT login API Direct email/password authentication for first-party REST endpoints.
JSON API
These endpoints are not part of the OAuth2/OIDC flow. They issue an HS256 API access token and an HTTP-only refresh cookie.
POST /auth/login Authenticate JSON email/password credentials.
POST /auth/refresh Rotate the refresh cookie and issue a new access token.
GET /auth/me Return the current API user from the bearer token.
POST /auth/logout Invalidate the current refresh token cookie.

Login

curl -X POST http://localhost:8080/auth/login \
  -H "Content-Type: application/json" \

    "email": "user@example.com",

Common failure modes Start with exact redirect URIs, PKCE state, and token audience.
Errors
invalid_client

Wrong client ID, unsupported client authentication, or missing registration.

invalid_grant

The code expired, was already used, or does not match the PKCE verifier.

invalid_redirect_uri

The redirect URI does not exactly match the registered value.

403 on POST /login

The Spring Security CSRF token is missing or stale.

401 from /auth/*

Credentials, refresh cookie, or bearer token are missing or invalid.
